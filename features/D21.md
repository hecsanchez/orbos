# D21 — Deploy Configuration

## Summary

Production deployment configs for all apps: API via Docker (Railway/Render), Next.js dashboards via Vercel, CI/CD via GitHub Actions.

## Files Created

### API Docker Build
- `apps/api/Dockerfile` — Multi-stage build (deps → build → production), Node 20 Alpine, pnpm via corepack
- `apps/api/.dockerignore` — Excludes node_modules, dist, .env, test files

### Railway
- `railway.json` — Dockerfile builder config, health check at `/api`, restart on failure (max 3 retries)

### Render
- `render.yaml` — Blueprint: web service (orbos-api) + PostgreSQL 16 (orbos-db), env vars for DATABASE_URL, LLM keys, R2 storage

### Vercel (Next.js apps)
- `apps/parent/vercel.json` — Build via `nx run parent:build`, pnpm monorepo install
- `apps/studio/vercel.json` — Build via `nx run studio:build`, pnpm monorepo install

### GitHub Actions CI
- `.github/workflows/ci.yml` — 5 parallel jobs on push/PR to main:
  - Typecheck (api)
  - Unit tests (vitest)
  - Docker image build (API)
  - Next.js build (parent)
  - Next.js build (studio)

## Deploy Targets

| App | Platform | Trigger |
|-----|----------|---------|
| API | Railway or Render | Push to main (auto) |
| Parent Dashboard | Vercel | Push to main (auto) |
| Curriculum Studio | Vercel | Push to main (auto) |
| iPad | Expo EAS | Manual (`eas build`) |

## Environment Variables (API)

Required in production:
- `DATABASE_URL` — PostgreSQL connection string
- `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` — LLM providers
- `R2_ENDPOINT`, `R2_ACCESS_KEY`, `R2_SECRET_KEY`, `R2_BUCKET` — Cloudflare R2
- `LLM_PROVIDER`, `LLM_MODEL` — Model selection
