# TASK: Day 17 — Curriculum Studio

You are a web engineering agent building Orbos. Read `context.md` and `CLAUDE.md` at the root before writing any code.

`apps/studio/` is an Nx Next.js boilerplate. Build an internal admin tool with 4 sections: lesson approval queue, phenomenon approval queue, safety log viewer, standards browser. No auth for pilot.

Rewrite `apps/studio/src/app/layout.tsx`:
- Sidebar navigation: Inicio, Lecciones, Fenómenos, Seguridad, Estándares
- Header "Orbos Curriculum Studio" in sidebar
- Dark sidebar (`bg-slate-900`) with light text, hover states
- Main content area fills remaining width

Modify `apps/studio/src/app/global.css`:
- Tailwind directives + admin palette CSS variables

## Step 4 — Lesson Approval Queue

Create `apps/studio/src/app/lessons/page.tsx`:
- Table of lesson scripts with expandable rows
- Columns: standard_id, safety status badge, admin status badge, created date
- Filter dropdown: All / Pending / Approved
- Expand row: renders script blocks via `LessonPreview` component
- Approve button on pending lessons → `PUT /lessons/:id/approve`

Create `apps/studio/src/components/LessonPreview.tsx`:
- Renders `InteractionBlock[]` as readable cards

## Step 5 — Phenomenon Approval Queue

Create `apps/studio/src/app/phenomena/page.tsx`:
- List proposals grouped by student (name, age, grade)
- Each proposal: title, linked_standards, status badge
- Expand: facilitation guide via `FacilitationGuidePreview`, evidence_prompt, materials_needed
- Approve button → `PUT /phenomena/:id/approve`

Create `apps/studio/src/components/FacilitationGuidePreview.tsx`:
- Parses JSON facilitation guide string
- Renders: overview, daily steps with discussion prompts, materials, success indicators
- Graceful fallback for unparseable guides

Create `apps/studio/src/components/StatusBadge.tsx`:
- Colored pill badges: pending (yellow), approved (green), completed (blue)
- Also supports boolean pass/fail display
