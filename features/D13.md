## Step 1 — Profile Store

Create `apps/ipad/src/stores/profile.store.ts` using Zustand (check if already installed — if not, install it: `pnpm add zustand -w`):

```typescript
export interface LocalProfile {
  id: string;              // matches student.id from API
  name: string;
  age: number;
  grade_target: number;
  interests: string[];
  avatar_id: string;       // references one of the 8 avatar options
  accent_color: string;    // hex color, unique per profile
  created_at: string;
}

export interface CreateProfileInput {
  name: string;
  age: number;
  avatar_id: string;
}

export interface ProfileStore {
  profiles: LocalProfile[];
  activeProfile: LocalProfile | null;

  // Loads profiles from SQLite on app start
  loadProfiles(): Promise<void>

  // Creates profile in API + stores locally in SQLite
  addProfile(input: CreateProfileInput): Promise<LocalProfile>

  // Sets the active session profile
  setActiveProfile(profile: LocalProfile): void

  // Clears active profile — called on session end
  clearActiveProfile(): void

  // Syncs profile list with API in case of changes from another device
  syncProfiles(): Promise<void>
}
```

Profiles are stored in a new SQLite table. Add it to `apps/ipad/src/db/schema.ts`:

```sql
CREATE TABLE IF NOT EXISTS profiles (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  age INTEGER NOT NULL,
  grade_target INTEGER NOT NULL,
  interests TEXT NOT NULL,   -- JSON array stored as string
  avatar_id TEXT NOT NULL,
  accent_color TEXT NOT NULL,
  created_at TEXT NOT NULL,
  synced INTEGER NOT NULL DEFAULT 0
);
```

Requirements:
- `grade_target` is derived automatically from age on creation — do not ask the child:
  - age 5–6 → grade 1–2
  - age 7–8 → grade 3–4
  - age 9–11 → grade 5–6
- `interests` default to age-appropriate values — real interest selection is post-MVP:
  - young (5–6): `['animales', 'colores', 'música']`
  - middle (7–8): `['dinosaurios', 'deportes', 'naturaleza']`
  - older (9–11): `['tecnología', 'arte', 'ciencia']`
- `accent_color` is assigned automatically from a predefined map keyed by `avatar_id` — not user-selectable
- On `addProfile`: call `POST /students` first, use the returned `id` as the profile `id`, then store locally
- On `syncProfiles`: call `GET /students`, merge with local — local wins for avatar/accent_color since those are device-only fields

**Navigation:**
- Back button visible — this is the only screen in the app with a back button
- Back returns to profile selector without saving
