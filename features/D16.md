# TASK: Day 16 — Parent Dashboard

You are a web engineering agent building Orbos. Read `context.md` and `CLAUDE.md` at the root before writing any code.

`apps/parent/` is an Nx Next.js boilerplate. Build a read-only dashboard showing per-child mastery overview, SEP coverage, and evidence portfolio. No auth for pilot.

---

## Step 1 — Setup

Modify `apps/parent/package.json` — add `@orbos/types: "workspace:*"`

Modify `apps/parent/src/app/layout.tsx`:
- Set title "Orbos - Panel de Padres"
- Set `lang="es"` on html element
- Add base body styling classes

Modify `apps/parent/src/app/global.css`:
- Tailwind directives + Orbos color CSS variables (`--orbos-primary`, `--orbos-accent`, etc.)
- Remove NX boilerplate styles

---

## Step 2 — API Client

Create `apps/parent/src/lib/api.ts`:
- `API_URL` from `NEXT_PUBLIC_API_URL` env var (default `http://localhost:3000`)
- Functions: `getStudents()`, `getMasteryForStudent(studentId)`, `getEvidenceForStudent(studentId)`, `getPhenomenaForStudent(studentId)`, `getStandards(grade?)`
- All use `fetch` with `cache: 'no-store'`

---

## Step 3 — Student Selector (Home Page)

Rewrite `apps/parent/src/app/page.tsx`:
- Fetch students on mount
- Student cards: name initial avatar, name, age, grade, interests
- Click navigates to `/student/[id]`
- Empty state if no students
- Loading and error states

Create `apps/parent/src/components/StudentCard.tsx`:
- Reusable card with Link to student dashboard
- Shows initial, name, age, grade label, interests

---

## Step 4 — Student Dashboard

Create `apps/parent/src/app/student/[id]/page.tsx`:
- Three sections:
  1. **MasteryHeatMap**: grid of standards per subject, cells colored by mastery_level (gray→red→yellow→green)
  2. **CoverageChart**: horizontal progress bars per subject showing % at mastery ≥ 0.8
  3. **EvidencePortfolio**: evidence grouped by phenomenon, photos as thumbnails, audio as players
- 5-minute auto-refresh via `setInterval`
- Back navigation to student selector
- Fetches mastery, evidence, phenomena, and standards in parallel

---

## Step 5 — Components

Create `apps/parent/src/components/MasteryHeatMap.tsx`:
- Groups standards by subject
- Color cells using HSL interpolation: gray (no data) → red (low) → yellow (mid) → green (mastered)
- Tooltip on hover showing standard ID, description, and mastery percentage
- Legend row at bottom

Create `apps/parent/src/components/CoverageChart.tsx`:
- Calculates mastered (≥ 0.8) count per subject
- CSS progress bars with color based on percentage
- Shows mastered/total and percentage

Create `apps/parent/src/components/EvidencePortfolio.tsx`:
- Groups evidence by phenomenon_id
- Photo evidence as image thumbnails
- Audio evidence with native `<audio>` players
- Empty state message when no evidence

- Student selector shows students, click navigates to dashboard

- Coverage chart shows progress bars per subject
