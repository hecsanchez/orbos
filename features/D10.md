Architecture First — Interaction Renderer
Before building any component, create the renderer that sequences them.
Create apps/ipad/src/components/InteractionRenderer.tsx:
typescriptinterface InteractionRendererProps {
script: InteractionBlock[];
studentId: string;
standardId: string;
onComplete: (attempts: AttemptResult[]) => void;
}
The renderer:

Receives the full script array
Shows one block at a time — never two simultaneously
Tracks the current index internally
On each block completion, records the attempt result (correct, time_spent, hint_used)
When all blocks are complete, calls onComplete with the full attempts array
TTS speaks the tts_text of each block automatically on mount — no child action required to hear the instruction
Never shows a back button — child can only move forward

Requirements:

Continue button activates after TTS completes or after 2 seconds (whichever comes first) — never block the child indefinitely
This component never records a correct/incorrect attempt — it is always correct by completion
Tap anywhere on the card (not just the button) also triggers continue after TTS completes
Match the design exactly — title, body text, continue button placement and style


Component 2 — multiple_choice
Create apps/ipad/src/components/interactions/MultipleChoice.tsx:
typescriptinterface MultipleChoiceProps {
question: string;
tts_text: string;
options: { id: string; label: string; correct: boolean }[];
hint_text?: string;
onComplete: (result: { correct: boolean; hint_used: boolean; time_spent_seconds: number }) => void;
}
Requirements:

TTS reads tts_text on mount before child can interact
Options are tappable after TTS completes
Correct answer: green highlight + continue automatically after 1.5 seconds
Wrong answer: neutral grey highlight + gentle bounce animation +
options remain tappable. Never use red. Never show a negative
message. After wrong answer, TTS says something encouraging like
"Inténtalo de nuevo" before child can re-select.
After 2 wrong attempts on the same question: show hint button if hint_text is provided
Tapping hint reads hint_text via TTS and sets hint_used: true in the result
Timer starts when options become tappable — used for time_spent_seconds
Only one option selectable at a time — tapping a new option deselects previous
Options order is randomized on mount — never show in the same order as the script


Component 3 — drag_drop
Create apps/ipad/src/components/interactions/DragDrop.tsx:
typescriptinterface DragDropProps {

}
Requirements:

TTS reads tts_text on mount
Items are draggable after TTS completes
Use React Native's PanResponder or react-native-gesture-handler — check which is available in Expo and use the appropriate one
Dragging: item follows finger smoothly, snaps back if dropped outside a target
Correct drop: item snaps into target, target highlights green, item locked in place
Wrong drop: item returns to original position with a gentle bounce.
No color change on the target. No negative sound or animation.
All items must be placed before onComplete fires
If all placements are correct: correct: true
If any placement is wrong at the end: correct: false
Hint button appears after 30 seconds of no correct placement


Component Registration
Create apps/ipad/src/components/interactions/index.ts — the component registry:
typescriptexport const INTERACTION_COMPONENTS = {
story_card: StoryCard,
multiple_choice: MultipleChoice,
drag_drop: DragDrop,
} as const;

export type ComponentName = keyof typeof INTERACTION_COMPONENTS;
The InteractionRenderer uses this registry to look up components by name. If a component name is not in the registry, it throws a visible error — never silently skip a block.
Update packages/types/src/index.ts with the shared interaction types:
