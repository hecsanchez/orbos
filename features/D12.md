Today you build the two most critical pieces of the iPad app: the Session Engine that orchestrates a full learning session, and the offline cache that makes the entire session work in airplane mode. These two are built together because they are deeply coupled — the Session Engine reads from the cache, not the network.

Architecture Overview
Session start (requires network):
1. Fetch daily plan from API
2. For each lesson/practice item, fetch script by lesson_script_id
3. Store plan + all scripts in SQLite
4. Session begins — device can go offline

Mid-session (fully offline):
1. Session Engine reads plan from SQLite
2. Renders each item using cached scripts
3. Writes attempts to SQLite attempt_queue
4. Evidence files stored locally

Session end / reconnect:
1. Flush attempt_queue to API
2. Trigger mastery recalculation
3. Fetch next day's plan (if online)
